# **Use Case UC-001:**  

1. # **Executive Summary**

## Overview

A **hybrid quantum-classical recursive algorithm** addresses the NP-hard **sensor placement problem in water distribution networks** (WDNs). The approach combines graph preprocessing, quantum Maximum Weighted Independent Set (MWIS) computation, coarsening reduction, and classical post-processing to select optimal sensor locations.

| Implementation strategy |
| :---- |
| *Summarise the implementation strategy methodology used for the simulations, including any bottlenecks encountered and the refinements or mitigation strategies implemented to address them.* |
| Methodology **Platform**: Pasqal cloud emulator (DGX A100 GPU cluster). Simulations adhered to QPU physical limitations (atomic confinement, sequence duration, detuning/Rabi frequency bounds) to ensure hardware transferability. **Algorithm**: A **Divide, Solve, Contract** recursive strategy: **Divide**: Spectral clustering partitions graphs into QPU-compatible subgraphs (\~30 nodes) with halo extensions for boundary consistency **Solve**: Sequential quantum MWIS computation using three detuning strategies (Local, DMM, Global) with forced activations across partition boundaries **Contract**: Spatially-aware coarsening reduces graph size using distance-decay functions to preserve geometric relationships The process repeats until convergence (typically ≤20 nodes) and a centroid-anchored unfolding step projects solution back with cardinality adjustment. *See main.tex Sections 1.3-1.6 (Implementation Strategy, Data Preparation, Overview and Theoretical Description)* Bottlenecks and Solutions 1\. State-of-the-Art Detuning Failure **Problem**: Standard approach failed on realistic graphs—ignored cumulative van der Waals effects, lacked upper bounds, produced invalid solutions. **Solution**: **Local Detuning Method** accounts for cumulative interactions with explicit bounds. **DMM (Detuning Map Modulation)** provides hardware-feasible approximation. Both extend to MWIS via weight-dependent biasing. Note however that the current QPU hardware only admits global detuning. Emulator is the only option for local detuning and DMM.  *See main.tex Sections 1.11.1-1.11.5 (State-of-the-art Approach, Theoretical Local Approach: MIS/MWIS, Real Use Case DMM Approach: MIS/MWIS)* 2\. Hardware Capacity Constraints **Problem**: QPU limited to \~30 nodes per partition; strict atomic confinement (38 µm radius, 5 µm spacing), 6000 ns sequence duration. **Solutions**: Conservative partitioning with halo extensions, register mapping optimization for pseudo-unit disk graphs, multiple implementation pathways (Local/DMM/Global). *See main.tex Sections 1.8-1.9 (Pasqal Emulator and QPU Constraints, Topological Constraints)* 3\. Topological Embedding **Problem**: Strict pseudo-unit disk graph requirements; certain configurations unembeddable. **Solution**: Fruchterman-Reingold force-directed layout transformation enables real-world network topology embedding. *See main.tex Section 1.10 (Pseudo Unit Disk Graph Transformation)* 4\. Sequential Processing Consistency **Problem**: Maintaining global MWIS consistency across partitioned subgraphs. **Solution**: Halo extension mechanism with forced activations ensures global independence property across partition boundaries. Coarsening procedure with spatially-aware importance transfer and geometric centroid updating ensures that the coarsened graph preserves both the weight distribution and topological properties. *See main.tex Section 1.6.2 (Sequential Solving with Halo Extension) and 1.6.3 (Coarsening Reduction)* Summary Hybrid quantum-classical recursive algorithm with Divide, Solve, Contract approach. Key innovations: local detuning method and DMM approximation overcome state-of-the-art limitations; halo extensions ensure sequential consistency; spatially-aware coarsening preserves topological properties. Methodology strictly adheres to QPU hardware constraints while enabling scalable processing. *See main.tex Section 1.3 (Implementation Strategy)* |

| 2\. Results & benchmark |
| :---- |
| *Summarise the main results of the simulations and how they compare against classical state of the art benchmarks.* |
| Benchmark Setup We have performed two distinct benchmarks: Coarsening and unfolding heuristics vs classical selection of k sensors Quantum MWIS simulation (4 methods) vs classical MWIS Coarsening:  80 small synthetic graphs (up to 20 nodes). Coarsening/Unfolding heuristic validation.  **Our algorithm is near-optimal** (0.015378), outperforming the naive approach (0.037448) and far from the worst case (0.165401). *See Section 1.6.5 (Evaluation and Benchmarking)*   Simulation: 15 benchmark instances from real-world EPANET water network data. Four quantum detuning strategies (Local, DMM, Global, State-of-the-art) compared against classical brute-force solutions. All simulations on Pasqal cloud emulator with QPU constraints. **Performance Metrics**: **Exactitude Probability** (probability of measuring exact brute-force optimum) and **Accuracy Ratio** (average closeness to brute-force optimum). *See main.tex Section 2 (Results)* Performance Hierarchy **Local Detuning** and **DMM**: Consistently outperform other methods. DMM closely approximates local performance and sometimes exceeds it. Successfully handle irregular graphs up to 32 nodes. **Global Detuning**: Adequate when graph topology exhibits low detuning variance; degrades significantly on complex graphs. **State-of-the-Art**: Frequently achieves zero exactitude on realistic graphs, producing invalid solutions that violate independence constraints. Works only on symmetric simple graphs. *See main.tex Sections 2.2 and 2.6 (Advantages of the Local Detuning Approach, Benchmarking MIS Approaches)* Comparison with Classical Baselines **Solution Quality**: Local and DMM methods match or closely approximate brute-force optimum. Both quantum and classical approaches identify the same solution sizes (e.g., MIS of 16 nodes on a 32-node instance). Discrepancies arise from multiple valid solutions, not quality differences. **State-of-the-Art Failure**: Baseline quantum approach produces invalid solutions on realistic graphs, confirming the necessity of advanced detuning computation. **Runtime**: No runtime or resource advantage claimed over classical methods at this stage. Proof-of-concept demonstrates feasibility rather than speedup. *See main.tex Sections 2.3-2.4 (MIS Comparative Example, Scalability Evaluation) and Section 3.1 (Results and Learnings)* Key Findings Weight-dependent biasing essential for high-quality MWIS solutions on neutral-atom quantum systems DMM provides hardware-feasible alternative approximating theoretical local detuning State-of-the-art fails on realistic graphs, confirming need for advanced methods Consistent performance across diverse graph topologies validates approach *See main.tex Section 2.6-2.7 (Benchmarking MIS/MWIS Approaches)* Summary Quantum-classical hybrid approach successfully produces high-quality MWIS solutions on realistic water network topologies. Local and DMM methods consistently outperform state-of-the-art quantum baseline. Advanced detuning strategies accounting for cumulative interactions and weight-dependent biasing are essential for practical quantum optimization on neutral-atom systems. *See main.tex Section 2 (Results)* |

| 5\. Justifications for implementation on QPUs |
| :---- |
| *Summarise the justification of the feasibility/readiness for implementation on quantum processing units, and advance to Phase 4 with the support of OQI.* |
| Overview Phase 3 noiseless emulator simulations validated the quantum-classical hybrid approach on irregular graphs up to 32 nodes while strictly adhering to actual QPU instrumental limitations, confirming readiness to noisy emulation and QPU hardware execution in Phase 4\. *See main.tex Section 5 (Moving to Phase 4: Noisy Emulation and QPU Hardware Implementation)* Justification for QPU Implementation Feasibility **Hardware Constraint Compliance**: All simulations adhered to QPU physical limits (atomic confinement, qubit capacity, sequence duration, control parameter bounds). Every tested configuration is directly transferable to actual QPU devices. **Proven Performance**: High-quality solutions on 15 real-world benchmark instances validate meaningful results under realistic conditions and hardware constraints. **Tiered Implementation**: Three pathways provide immediate-to-future deployment options: Global detuning (immediately available; baseline) DMM (near-term QPUs; superior performance) Local detuning (future optimal) This tiered approach enables incremental deployment as QPU hardware capabilities evolve, ensuring immediate applicability and future extensibility.  *See main.tex Sections 1.7, 1.8, 2.6, and 5.1* Phase 4 Objectives **Transition Path**: Noiseless emulator (Phase 3\) → Noisy emulation → QPU hardware execution (Phase 4\) **Noisy Emulation**: Model realistic noise/decoherence; insights into performance degradation; optimize for noisy conditions before QPU execution **QPU Hardware**: Multi-step algorithm on actual analog QPUs; direct noise/decoherence/imperfection handling; adapt implementation pathways (global, DMM, local) to hardware characteristics *See main.tex Section 5.2* Implementation Strategy **Recommended Sequence:** Global approach (immediate baseline; adequate performance for low-variance graphs) DMM (when hardware-available; pragmatic with superior performance) Noise characterization (quantify impact; compare QPU vs. emulator) Iterative execution (full recursive algorithm under noisy conditions) **Success Criteria**: Noisy emulation: Accuracy \>0.80, Exactitude \>0.50 (20-30 nodes) QPU hardware: Accuracy \>0.75, Exactitude \>0.40 (20-30 nodes) Performance degradation quantified Scalability validated within current QPU constraints *See main.tex Sections 5.3 and 5.4* Scope Clarification **Phase 4 Focus**: Noisy emulation and QPU hardware execution; benchmarking under realistic noise; performance characterization **Deferred to Medium-Term**: Leak detection effectiveness assessment (WNTR); NRW reduction quantification; leak detection validation loop integration *See main.tex Sections 5.1 and 5.2* Summary Strong justification for QPU implementation: methodology adheres to actual QPU constraints, demonstrates high-quality solutions on realistic topologies, provides implementation-ready algorithm with multiple pathways. Phase 4 bridges noiseless feasibility to realistic hardware performance with clear success criteria. Technical validation, implementation readiness, and clear objectives support OQI advancement to hardware validation. *See main.tex Section 5* |

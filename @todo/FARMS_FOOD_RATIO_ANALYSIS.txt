================================================================================
WHY FARMS/FOOD RATIO MATTERS
================================================================================

Constraint Tightness Example (6 food families, 3 periods):

LOW RATIO (HARD):
  5 farms / 6 foods = 0.83 farms/food
  -> Each food competes for only 5 farms x 3 periods = 15 slots
  -> Each farm-period can have <= 2 foods (one-hot constraint)
  -> Very tight allocation problem
  -> Weak LP relaxation
  -> Gurobi struggles

HIGH RATIO (EASY):
  50 farms / 6 foods = 8.33 farms/food
  -> Each food has 50 farms x 3 periods = 150 slots available
  -> Much more flexibility
  -> Strong LP relaxation
  -> Gurobi solves at root node

================================================================================
COUNTER-INTUITIVE FINDINGS
================================================================================

1. TOTAL AREA is NOT the main factor
   - Both 5 farms (100 ha) and 25 farms (13 ha) are HARD
   - 50 farms (100 ha) is EASY
   -> Area alone doesn't predict hardness

2. VARIABILITY (CV) is NOT significant
   - HARD: CV = 0.872
   - EASY: CV = 0.841
   -> Only 4% difference

3. MORE VARIABLES doesn't mean HARDER
   - 5 farms (90 vars) = TIMEOUT
   - 100 farms (1800 vars) = EASY in 2.8s
   -> Problem size is NOT hardness!

4. AREA PER VARIABLE is HIGHER for HARD problems
   - Counter-intuitive: more space per variable should be easier
   - But constraint structure matters more than space
